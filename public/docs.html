API-Dokumentation

Öffentliche REST-API für Episodendaten von Kack & Sachgeschichten.

https://kack.li
Authentifizierung: GET-Endpunkte sind öffentlich zugänglich (100 Anfragen / 15 Minuten pro IP). Admin-Endpunkte (POST) erfordern einen Bearer-Token im Authorization-Header: Authorization: Bearer <ADMIN_TOKEN>
Öffentliche Endpunkte
GET /api/episodes public
Gibt eine paginierte Liste aller Episoden zurück. Unterstützt Volltextsuche sowie Filterung nach Gast oder Thema.
ParameterTypBeschreibung
q optionalstringVolltextsuche in Titel und Beschreibung
guest optionalstringFilter nach Gastname
topic optionalstringFilter nach Thema
limit optionalintegerAnzahl Ergebnisse (Standard: 24, max: 100)
offset optionalintegerOffset für Paginierung (Standard: 0)
{ "total": 392, "limit": 24, "offset": 0, "episodes": [ { "id": 383, "title": "#008: Independence Day 2 – Die Kamikaze-Scheibe", "film_title": "Independence Day: Wiederkehr", "pub_date": "Mon, 01 Aug 2016 10:11:45 +0200", "duration": "1:23:45", "episode_num": 8, "guests_json": "[\"Max Mustermann\"]", "topics_json": "[\"Science-Fiction\", \"Blockbuster\", \"Weltuntergang\"]", "chapters_json": "[{\"time\":\"00:05:00\",\"title\":\"Intro\"}]", "parsed_at": "2026-04-08T10:00:00.000Z", ... } ] }
GET /api/episodes/:id public
Gibt eine einzelne Episode mit allen Feldern zurück.
GET /api/guests public
Liste aller Gäste, sortiert nach Anzahl der Auftritte.
[ { "name": "Dag von SDP", "count": 12 }, { "name": "Alea (Saltatio Mortis)", "count": 3 }, ... ]
GET /api/topics public
Liste aller Themen, sortiert nach Häufigkeit.
[ { "name": "Science-Fiction", "count": 47 }, { "name": "Gaming", "count": 38 }, ... ]
GET /api/status public
Gibt den aktuellen Status der Datenbank zurück.
{ "episodes": 392, "parsed": 392, "last_sync": "2026-04-08T09:00:00.000Z", "openai": true }
Admin-Endpunkte (Bearer-Token erforderlich)
POST /api/sync admin
Synchronisiert den RSS-Feed und fügt neue Episoden zur Datenbank hinzu.
{ "ok": true, "count": 392, "new": 2, "synced_at": "2026-04-08T09:00:00.000Z" }
POST /api/episodes/:id/parse admin
Analysiert eine einzelne Episode mit OpenAI und extrahiert Kapitel, Gäste, Themen und Film-Titel.
POST /api/parse-all admin
Analysiert alle noch nicht geparsten Episoden im Hintergrund (~3 req/s).
ParameterTypBeschreibung
force optional1Alle Episoden neu analysieren (auch bereits geparste)
